### 3.2 SyncService.OnFileChanged `async void` + catch-all 吞异常

**问题**:[SyncService.cs:114-137](../src/OmniKeyVault.Application/Services/SyncService.cs) `private async void OnFileChanged(...)` 是 `async void`(只能用 fire-and-forget 事件处理器),但内部 `catch { /* merge failure should not crash the watcher */ }` 吞噬所有异常。

**影响**:
- 同步合并失败时无日志、无 UI 提示、无错误退出码
- 用户不知道"自动同步"是否真的工作
- 调试困难(异常被静默丢弃)

**建议**:
1. 改 `async Task OnFileChangedAsync(...)`,事件处理器内 `_ = OnFileChangedAsync(...)` + `.ContinueWith(t => LogException(t.Exception), OnlyOnFaulted)`
2. catch 块按异常类型分类:`CryptoException`(密文损坏)→ 日志 + UI toast;`IOException`→ 重试 3 次;`SyncConflictException`→ 触发 GUI 冲突向导
3. 同步失败计数器,连续 5 次失败暂停 watcher + 提示用户

**预估**:4h

### 3.3 Serilog 日志脱敏过滤器未实现

**影响**:
- 日志若误含敏感字段值(如 Debug 级别输出 Entry 字段),会落盘到 `%LOCALAPPDATA%\OmniKeyVault\logs\okv-{date}.log`

**建议**:

### 3.4 CI/CD 自动化缺失

```yaml
      - run: dotnet test OmniKeyVault.sln -c Release --no-build --logger trx
      - uses: actions/upload-artifact@v4
        if: always()
        with: { name: trx, path: '**/TestResults/*.trx' }
  benchmark:
    runs-on: windows-latest
    needs: build-test
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-dotnet@v4
      - run: dotnet run --project tools/OmniKeyVault.Benchmark -c Release
```

**预估**:1d(含 Windows / Linux matrix + 发布产物上传)

#### 4.1.1 WebAuthn / FIDO2 二次因子

**价值**:[SECURITY.md §3.1](./SECURITY.md) 已预留"WebAuthn / FIDO2 私钥派生恢复码"选项,但未实现。可用作:

**实现路径**:
1. 引入 `Fido2NetLib` NuGet 包

**价值**:[SECURITY.md §12.1](./SECURITY.md) 已规划,但需 v1.0 发布后启动。

**实现路径**:
1. `OKV2` magic + 混合加密(XChaCha20-Poly1305 + ML-KEM-768)
2. ML-KEM 用 `BouncyCastle.Cryptography` 8.x 包(已含 NIST FIPS 203 标准)
3. v2 客户端可读 v1(.okv 头有 version 字段),自动升级;v1 客户端遇 v2 提示升级
4. 不改 KDF(Argon2id 仍抗量子,量子对 KDF 无显著优势)

**预估**:8 周(含算法选型 / 互操作测试 / 性能基准)

#### 4.1.3 SSH 私钥管理 + ssh-agent 集成

**预估**:1 周

1. Blazor Server(SignalR 实时)+ WebAuthn 二次因子
2. Web 端不存密钥,通过 IPC 调用本地 `okv.exe`(仅当本机安装)
3. 远程场景:用户上传 `.okv` 文件 + 输入主密码 → 浏览器内 Argon2id + XChaCha20-Poly1305(用 `libsodium.js`)
4. **风险**:浏览器侧信道大,建议仅 dev Profile

**预估**:12 周

#### 4.2.4 TUI 模式(终端 UI)

**价值**:服务器 / SSH 场景下,无 GUI 但需交互式查看(优于 CLI 一次性命令)。

**实现路径**:
1. 引入 `Terminal.Gui` 包(.NET TUI 框架)
2. 复用 `OmniKeyVault.Application` 全部服务
3. 三栏布局:Profile 列表 / Entry 列表 / 详情

### 4.4 用户体验方向

#### 4.4.1 全局热键解锁 + 自动填充

**价值**:用户在任何应用中按 `Ctrl+Shift+U` → 弹出解锁 → 自动填充当前焦点字段。

**实现路径**:
1. Windows:注册全局热键(`RegisterHotKey` Win32 API)
2. macOS:`CGEventTap`(需辅助功能权限)
3. Linux:X11 `XGrabKey`
4. 自动填充:Windows用 `SendInput`;macOS 用 `CGEventCreateKeyboardEvent`;Linux 用 `xdotool`

**预估**:2 周(跨平台占大头)

#### 4.4.2 密码健康度报告

**价值**:用户主动检查 vault 健康度。

**功能**:
- 弱密码(zxcvbn 评分 < 3)
- 重复密码(同一字段值跨 entries)
- 过期 entries(`expires_at < now`)
- 长期未更新(> 1 年)
- 泄露检测(§4.1.5 HIBP 集成)

**实现路径**:
1. 新建 `HealthReportService`(Application 层)
2. GUI:`HealthReportWindow`(列表 + 详情 + 一键修复)
3. CLI:`okv health report`

**预估**:1 周

#### 4.4.3 审计日志(本地)

**价值**:[SECURITY.md](./SECURITY.md) 当前无审计日志,用户无法追溯"谁在何时操作了什么"。

**实现路径**:
1. 新建 `AuditLogService`(Application 层)
2. 记录:unlock / lock / create-entry / update-entry / delete-entry / rotate / change-password / sync
3. 持久化:`%APPDATA%/OmniKeyVault/audit.log`(明文,本地)
4. GUI:`AuditLogWindow`(列表 + 筛选 + 导出)

**预估**:1 周

#### 4.4.4 紧急联系人 / 信任恢复

**价值**:用户忘记主密码 + 丢失 Recovery Key 时,可由信任的紧急联系人协助恢复。

**实现路径**:
1. 用户指定 N 个紧急联系人(每个一个 Ed25519 公钥)
2. 用 Shamir 秘密共享(M-of-N)分片主密码 KEK
3. 联系人用自己的私钥解片;M 片聚合可重建 KEK
4. 延时解锁:联系人确认后等待 X 天(可配置),期间用户可取消

**预估**:3 周

#### 4.4.5 主题切换(暗色 / 浅色 / 跟随系统)

**价值**:[INTERNAL.md §5.7](./INTERNAL.md) 已有 `ui.theme` 配置,但 GUI 未实现。

**预估**:3 天

#### 4.4.6 多语言扩展(ja-JP / ko-KR)

**价值**:当前 zh-CN / en-US,日韩用户群可扩展。

**预估**:每语言 1 周(翻译 + 测试)

### 4.5 企业功能方向

#### 4.5.1 团队共享(零知识 + 安全聚合)

**价值**:[SECURITY.md §12.4](./SECURITY.md) 当前不支持;但企业市场需要。

**实现路径**:
1. 团队 KEK 包装(每个用户公钥包装一次团队 KEK)
2. 用户加入:管理员用用户公钥包装团队 KEK
3. 用户退出:重新生成团队 KEK + 所有条目重新加密(O(N))
4. 加密聚合服务器:只见密文,不知 KEK

**预估**:20 周

#### 4.5.2 RBAC + 审计日志转发

**价值**:企业合规(SOC2 / ISO 27001)。

**实现路径**:
1. 角色定义:Admin / Editor / Viewer
2. per-entry ACL
3. 审计日志转发:SIEM(Splunk / ELK)通过 Syslog

**预估**:16 周

#### 4.5.3 SSO 集成(SAML / OIDC)

**价值**:企业用户用 SSO 解锁(替代主密码)。

**风险**:[SECURITY.md §12.4](./SECURITY.md) 不支持,因为零知识模型硬约束。SSO 解锁意味着 KEK 由 SSO IdP 派生,违反"主密码永不出设备"。

**替代方案**:SSO 仅用于身份认证,**不**用于密钥派生;密钥派生仍需主密码或 WebAuthn。

**预估**:6 周
