## 1. 产品简介

OmniKey Vault 是一款**本地优先、端到端加密**的密码与密钥管理器。所有密码学操作在您的设备本地完成，主密码永不出设备。

### 核心特性

- **双重 Argon2id 密钥拉伸（v1.6 新增）**: 双轮 KDF 链（总计 320 MiB 内存成本），即使源码公开+金库文件泄露，离线爆破成本也翻倍
- **零知识加密**: 使用 Argon2id 密钥派生 + XChaCha20-Poly1305 AEAD 加密 + Ed25519 签名
- **自定义格式**: `.okv` 二进制格式，不被 hashcat / KeePass 等通用工具识别
- **多设备同步**: 支持 WebDAV 云同步、本地文件夹同步、S3 兼容存储同步（v2.0 新增），使用向量时钟解决冲突
- **多 Profile**: 生产 (prod)、开发 (dev)、测试 (test) 环境隔离
- **TOTP 支持**: 内置 OTP 验证码生成器
- **平台模板**: 内置 66 个模板，覆盖国内外主流平台（GitHub、OpenAI、AWS、腾讯云、华为云、百度千帆、通义千问、DeepSeek、支付宝、微信支付等）
- **自动锁定**: 空闲超时、系统锁屏、系统休眠自动锁定
- **剪贴板保护**: 复制后自动清空剪贴板
- **开源安全**: 源代码完全公开，接受社区审计，安全设计不依赖隐蔽
- **密码生成器（v2.0 新增）**: 自定义长度/字符集/排除易混字符 + 密码短语模式，Ctrl+G 快捷唤起
- **凭据泄露检测（v2.0 新增）**: HaveIBeenPwned k-anonymity 模式，主动检测密码是否已在已知泄露库中
- **紧急联系人恢复（v2.0 新增）**: Shamir 分片恢复机制，主密码拆分为 N 份分片，任意 K 份可还原
- **加密容器导出（v2.0 新增）**: 导出为独立加密文件（.okvx），输入独立密码，方便安全分享
- **X.509 证书管理（v2.0 新增）**: 解析 PEM/PFX 证书，显示到期时间、指纹、颁发者
- **SSH Agent 集成（v2.0 新增）**: 一键将 SSH 私钥加载到 Windows ssh-agent
- **文件夹与标签（v2.0 新增）**: 文件夹 CRUD + 标签系统（侧边栏标签面板，点击筛选）
- **批量操作（v2.0 新增）**: 批量编辑/删除/导出，Ctrl+点击多选
- **多格式导入（v2.0 新增）**: 1Password .1pux、KeePass KDBX 3/4、EnPass JSON、CSV（LastPass/Chrome/Edge/Firefox）、.env 文件
- **密码强度全库扫描（v2.0 新增）**: 一键扫描所有条目的密码强度
- **系统通知（v2.0 新增）**: 条目过期等事件推送 Windows 系统通知
- **窗口位置记忆（v2.0 新增）**: 记住上次窗口位置和大小

## 2. 安装与卸载

### 2.1 系统要求

- Windows 10 1809 (17763) 或更高版本
- .NET 8 运行时（安装包已内置）

### 2.2 安装

1. 双击 `OmniKeyVault-Setup.exe` 运行安装程序
2. 选择安装目录（默认: `C:\Program Files\OmniKeyVault`）
3. 选择是否创建桌面快捷方式
4. 点击「安装」等待完成
5. 点击「完成」退出安装程序

### 2.3 卸载

- **方式一**: 打开「设置 → 应用 → 已安装的应用」，找到 OmniKey Vault，点击卸载
- **方式二**: 运行安装目录下的 `uninstall.exe`

> **注意**: 卸载不会删除您的金库文件 (`vault.okv`) 和配置文件。金库文件位于 `%USERPROFILE%\OmniKeyVault\`，配置文件位于 `%APPDATA%\OmniKeyVault\`。

### 2.4 首次启动

首次启动时会显示解锁窗口。如果没有现有金库，点击「创建新金库」按钮。

## 3. 创建保险库

### 3.1 创建步骤

1. 在解锁窗口点击「创建新金库」
2. **步骤 1 — 选择路径**: 选择金库文件的保存位置（默认: `%USERPROFILE%\OmniKeyVault\vault.okv`）
3. **步骤 2 — 设置主密码**:
   - 输入主密码（至少 8 个字符）
   - 确认主密码
   - 密码强度指示器会实时显示密码强度
   - **弱密码将被拒绝**（评分 < 2 的密码无法创建金库）
   - 建议使用 12+ 字符，包含大小写字母、数字和符号
4. **步骤 3 — 确认并保存**: 勾选「我已安全保存主密码」，点击「创建」
5. **步骤 4 — 恢复密钥**: 创建完成后会显示 192 字符的恢复密钥，请妥善保存

### 3.2 从云端拉取已有金库（v1.5 新增）

如果您已在其他设备上将金库同步到 WebDAV 云端，在新设备上无需创建新金库，可直接从云端拉取：

1. 在创建金库向导的欢迎页面，点击「☁ 从云端拉取已有金库 →」
2. 填写 WebDAV 配置：
   - **服务器地址**: 如 `https://dav.jianguoyun.com/dav/omnikey-vault/`
   - **用户名**: WebDAV 账号
   - **密码**: WebDAV 应用密码
   - **远端文件路径**: 如 `vault.okv`
   - **本地保存文件夹**: 选择本地保存位置
3. （可选）点击「🔗 测试连接」验证配置是否正确
4. 点击「拉取金库 →」
5. 拉取成功后，程序自动跳转到解锁窗口
6. 输入与原设备相同的主密码即可解锁

> **优势**: 跳过新建金库的完整流程（设置密码、生成恢复密钥等），直接使用云端已有金库登录。
>
> **注意**: 坚果云用户需在服务器地址中包含子目录路径（如 `/dav/omnikey-vault/`），不能直接使用根路径 `/dav/`。

### 3.3 主密码安全建议

| 强度 | 示例 | 建议 |
|------|------|------|
| 弱 | `password123` | 不要使用 |
| 中 | `MyCat2024!` | 仅用于测试 |
| 强 | `Tr0ub4dour&3#sky` | 可接受 |
| 很强 | `correct-horse-battery-staple-2026` | 推荐 |

> **警告**: 主密码是唯一保护您金库的密码。如果遗忘，**无法恢复**（即使恢复密钥也需要主密码来使用）。

### 3.4 恢复密钥

恢复密钥是一串 192 字符的随机字符串，在创建金库时生成。它的作用是：

- 在主密码遗忘时，可作为应急验证手段
- 请离线保存（如打印后锁在保险箱中）
- **切勿与金库文件保存在同一位置**

## 4. 解锁与锁定

### 4.1 解锁

1. 启动程序后显示解锁窗口
2. 确认金库路径（程序会自动记住上次打开的金库）
3. 输入主密码
4. 点击「解锁」或按 Enter 键
5. 解锁过程需要约 2-4 秒（v1.6 双重 Argon2id 密钥派生，总内存成本 320 MiB；v1 金库约 1-3 秒）

### 4.2 锁定

以下方式可以锁定金库：

- 点击顶部状态栏的「锁定」按钮
- 按 `Ctrl+L` 快捷键
- 空闲超时自动锁定
- 系统锁屏时自动锁定（可配置）
- 系统休眠时自动锁定（可配置）

锁定后，内存中的所有密钥和明文数据会被**立即清零** (`CryptographicOperations.ZeroMemory`)。

### 4.3 自动锁定

默认 15 分钟无操作后自动锁定。可在「设置 → 安全」中修改为 5/10/15/30/60 分钟。

## 5. 条目管理

### 5.1 新建条目

1. 点击左侧边栏的「+ 新建条目」按钮
2. 在编辑器窗口中填写：
   - **名称**: 条目名称（如 "OpenAI 生产环境"）
   - **平台**: 选择平台模板（GitHub / OpenAI / AWS / Stripe 等）
   - **字段**: 每个条目可包含多个字段，类型包括：
     - `密文` — 敏感信息（API Key、密码等），默认掩码显示
     - `文本` — 普通文本
     - `链接` — URL
     - `数字` — 数字
     - `日期` — 日期
     - `TOTP` — OTP 验证码 URI
     - `文件` — 文件引用
   - **标签**: 用空格分隔的标签（如 `prod critical`）
   - **备注**: 自由文本
   - **文件夹**: 可选，将条目归入文件夹
3. 点击「保存」

### 5.2 编辑条目

- 点击条目列表中的条目，然后点击「编辑」按钮
- 或右键点击条目，选择「编辑」

### 5.3 删除条目

- 右键点击条目，选择「删除」
- 确认后删除（不可撤销）

### 5.4 复制字段

- 点击条目详情中字段右侧的「⧉」按钮复制
- 或右键点击条目，选择「复制主字段」
- 复制后 8 秒自动清空剪贴板（可配置为 4/8/16/30 秒）

### 5.5 显示明文

对于 `密文` 类型的字段，点击「👁」按钮按住可临时显示明文，松开后自动掩码。

### 5.6 排序

点击顶部「排序」按钮可选择：
- 按名称 (A→Z)
- 按更新时间 (新→旧)
- 按创建时间 (新→旧)

## 6. 配置文件 (Profile)

OmniKey Vault 支持多配置文件 (Profile) 来隔离不同环境的密钥：

| Profile | 颜色 | 用途 | 横幅 |
|---------|------|------|------|
| `prod` | 绿色 | 生产环境 | 无 |
| `dev` | 黄色 | 开发环境 | "DEV — 非生产数据" |
| `test` | 蓝色 | 测试环境 | "TEST — 测试数据" |

### 6.1 切换 Profile

点击顶部状态栏的 Profile 标签，在弹出窗口中选择目标 Profile。

### 6.2 创建新 Profile

在设置窗口的「Profiles」部分，或通过种子导入功能创建新 Profile。

### 6.3 Profile 设置

每个 Profile 可独立配置：
- **参与同步**: 是否参与多设备同步
- **切换后自动锁定**: 切换 Profile 后立即锁定金库
- **空闲锁**: 空闲锁定时间

## 7. WebDAV 云同步

WebDAV 同步是 OmniKey Vault 的核心云同步功能，允许您在多台设备之间同步金库。

### 7.1 工作原理

```
设备 A (本地金库)                    WebDAV 服务器                  设备 B (本地金库)
     │                                   │                              │
     │── 1. 下载远端 .okv ──────────────→│                              │
     │←── 2. 返回远端 .okv ──────────────│                              │
     │                                   │                              │
     │   3. 向量时钟比较 + 条目合并       │                              │
     │   4. 保存合并后的本地 .okv         │                              │
     │                                   │                              │
     │── 5. 上传合并后的 .okv ──────────→│                              │
     │                                   │                              │
     │                                   │←── 6. 设备 B 下载 .okv ──────│
     │                                   │── 7. 返回 .okv ──────────────→│
     │                                   │                              │
     │                                   │←── 8. 设备 B 合并 + 上传 ────│
```

**关键安全保证**: WebDAV 服务器只存储**加密后的 `.okv` 文件**。即使服务器被入侵，攻击者也无法解密您的数据。

### 7.2 支持的 WebDAV 服务

以下服务均已测试兼容：

| 服务 | 服务器地址格式 | 备注 |
|------|---------------|------|
| Nextcloud / ownCloud | `https://your-server/remote.php/dav/files/username/okv/` | 需要应用密码 |
| Synology NAS | `https://your-nas:5006/okv/` | 需要 WebDAV Server 套件 |
| 坚果云 | `https://dav.jianguoyun.com/dav/okv/` | 需要应用密码（非登录密码） |
| Box | `https://dav.box.com/dav/okv/` | 需要应用密码 |
| 自建 WebDAV | `https://your-server/path/` | 需支持 RFC 4918 |

### 7.3 配置 WebDAV

1. 打开主界面，点击「⚙ 设置」
2. 找到「WebDAV 云同步」部分
3. 填写以下信息：
   - **服务器地址**: WebDAV 服务器的完整 URL
     - 例如: `https://dav.example.com/okv/`
     - URL 末尾建议加 `/`，远端文件路径会拼接到此 URL 后
   - **用户名**: WebDAV 账号用户名
   - **密码**: WebDAV 应用密码（**不是**您的登录密码）
     - 大多数 WebDAV 服务要求使用专用的应用密码
     - 坚果云: 在网页端「安全选项 → 添加应用」生成
     - Nextcloud: 在「设置 → 安全 → 设备和会话」生成
   - **远端文件路径**: 金库在服务器上的文件名
     - 默认: `vault.okv`
     - 如果多个金库使用同一服务器，可用不同文件名区分
4. 勾选「启用 WebDAV 同步」
5. 点击「🔗 测试连接」验证配置
   - 成功: 显示 "✓ 连接成功！服务器可访问。"
   - 失败: 显示具体错误信息
6. 点击「💾 保存配置」持久化设置

### 7.4 手动同步

配置完成后，在主界面顶部有三个同步按钮，分别对应不同的操作：

| 按钮 | 功能 | 使用场景 |
|------|------|----------|
| ⬇ 拉取云端 | 从 WebDAV 服务器下载并合并远端更改 | 在其他设备上编辑后，想在当前设备获取更新 |
| ⬆ 推送到云端 | 直接上传本地金库到 WebDAV 服务器 | 本地有更新，想同步到其他设备 |
| 🔄 本地同步 | 重新加载本地金库文件 | 从本地备份恢复或手动替换文件后刷新 |

**推荐使用方式**：
- **多设备协作**: 优先使用「⬇ 拉取云端」获取远端更新，编辑后使用「⬆ 推送到云端」同步
- **单设备备份**: 使用「⬆ 推送到云端」定期上传备份
- **本地恢复**: 使用「🔄 本地同步」加载手动复制的金库文件

同步结果会通过 Toast 提示：
- **无变化**: 远端和本地一致
- **拉取**: 从远端拉取了更新
- **推送**: 本地版本更新，已推送到远端
- **合并**: 双方都有更新，已自动合并
- **首次上传**: 远端不存在文件，已上传本地金库

### 7.5 自动同步

勾选「自动同步（解锁后拉取，保存后推送）」后：
- 每次解锁金库后，自动从 WebDAV 拉取最新数据
- 每次保存条目后，自动推送更新到 WebDAV

> **建议**: 在多设备使用场景下开启自动同步。在网络不稳定时，自动同步失败不会影响本地操作。

### 7.6 冲突解决

当两台设备同时修改了金库，同步时会检测到冲突。OmniKey Vault 使用以下策略：

1. **向量时钟比较**:
   - 本地领先 → 推送本地版本到远端
   - 远端领先 → 拉取远端版本到本地
   - 并发修改 → 执行条目级合并

2. **条目级合并规则**:
   - 新条目（仅一方有）→ 直接添加
   - 版本不同的条目 → 保留较高版本
   - 同版本不同内容 → **本地优先** (local-wins)，并提示冲突数量

3. **冲突提示**: 同步完成后会显示冲突数量。如需覆盖为远端版本，可使用本地文件夹同步的冲突解决向导。

### 7.7 首次使用 WebDAV 同步

**设备 A（已有金库）**:
1. 打开设置，配置 WebDAV
2. 点击「同步」按钮 → 首次上传金库到服务器

**设备 B（新设备）**:

**方式一：从云端拉取（v1.5 推荐）**:
1. 安装 OmniKey Vault
2. 在解锁窗口点击「创建新金库」
3. 在欢迎页面点击「☁ 从云端拉取已有金库 →」
4. 填写与设备 A 相同的 WebDAV 配置
5. 选择本地保存文件夹，点击「拉取金库」
6. 拉取成功后，输入与设备 A 相同的主密码解锁即可

**方式二：传统创建+同步**:
1. 安装 OmniKey Vault
2. 在解锁窗口点击「创建新金库」，**使用与设备 A 相同的主密码**
3. 打开设置，配置相同的 WebDAV 信息
4. 点击「同步」按钮 → 从服务器拉取金库
5. 程序会提示「请锁定后使用远端金库的密码重新解锁」
6. 锁定金库后重新解锁，即可看到设备 A 的所有条目

> **重要**: 两台设备必须使用**相同的主密码**创建金库，因为 KEK（密钥加密密钥）是从主密码派生的。如果主密码不同，远端文件的加密密钥不同，无法解密。
>
> **v1.2 新增**: 如果新设备创建的是空金库（无任何条目），同步时会自动拉取远端金库替换本地。如果本地金库已有数据且与远端不匹配，会提示「金库不匹配」错误，避免数据丢失。
>
> **v1.5 新增**: 在创建金库向导中直接从云端拉取已有金库，跳过新建流程，更简单快捷。

### 7.8 WebDAV 安全注意事项

1. **务必使用 HTTPS**: 确保服务器地址以 `https://` 开头，防止中间人攻击
2. **使用应用密码**: 不要使用登录密码，大多数服务支持生成专用的应用密码
3. **服务器只存密文**: 即使服务器管理员也无法看到您的明文数据
4. **定期检查设备列表**: 在设置中查看已注册设备，及时吊销不认识的设备
5. **备份金库文件**: 建议在本地保留金库文件的备份副本

## 9. S3 兼容存储同步（v2.0 新增）

除了 WebDAV 和本地文件夹同步，OmniKey Vault v2.0 新增了 S3 兼容存储同步功能，支持 AWS S3、MinIO、Cloudflare R2、阿里云 OSS 等对象存储作为同步后端。

### 9.1 工作原理

```
设备 A (本地金库)                    S3 兼容存储                  设备 B (本地金库)
     │                                   │                              │
     │── 1. 下载远端 .okv ──────────────→│                              │
     │←── 2. 返回远端 .okv ──────────────│                              │
     │                                   │                              │
     │   3. 向量时钟比较 + 条目合并       │                              │
     │   4. 保存合并后的本地 .okv         │                              │
     │                                   │                              │
     │── 5. 上传合并后的 .okv ──────────→│                              │
```

**安全保证**: S3 存储只保存加密后的 `.okv` 文件。使用 SigV4 签名认证，即使存储提供商被入侵，攻击者也无法解密您的数据。

### 9.2 支持的 S3 兼容服务

| 服务 | Endpoint 示例 | 备注 |
|------|---------------|------|
| AWS S3 | `https://s3.us-east-1.amazonaws.com` | 标准 S3 API |
| MinIO | `http://192.168.1.100:9000` | 自建对象存储 |
| Cloudflare R2 | `https://<account-id>.r2.cloudflarestorage.com` | 免出口费用 |
| 阿里云 OSS | `https://oss-cn-hangzhou.aliyuncs.com` | S3 兼容模式 |
| 腾讯云 COS | `https://cos.ap-guangzhou.myqcloud.com` | S3 兼容模式 |

### 9.3 配置 S3 同步

1. 打开「设置 → 同步」
2. 找到「S3 兼容存储同步」部分
3. 填写以下信息：
   - **Endpoint**: S3 服务的访问端点
   - **Region**: 区域（如 `us-east-1`）
   - **Bucket Name**: 存储桶名称
   - **Access Key**: 访问密钥 ID
   - **Secret Key**: 访问密钥 Secret
   - **Object Key**: 金库在存储桶中的对象键（默认: `vault.okv`）
4. 勾选「启用 S3 同步」
5. 点击「🔗 测试连接」验证配置
6. 点击「💾 保存配置」

> **注意**: Access Key 和 Secret Key 仅存储在本地配置中，不会上传到任何服务器。

## 16. 凭据泄露检测（v2.0 新增）

OmniKey Vault v2.0 集成了 HaveIBeenPwned 密码泄露检测服务，采用 k-anonymity 隐私保护模式。

### 16.1 工作原理

1. 对密码计算 SHA-1 哈希
2. 仅发送哈希值的前 5 个字符到 HaveIBeenPwned API
3. API 返回所有以这 5 个字符开头的哈希后缀及泄露次数
4. 本地比对完整哈希，确定密码是否已泄露

> **隐私保证**: 您的密码永远不会离开设备。仅发送 SHA-1 哈希的前 5 个字符，API 无法还原您的原始密码。

### 16.2 检测单个条目

1. 在条目详情中，点击密码字段旁的「🛡」按钮
2. 查看检测结果：
   - ✅ 安全：该密码未在已知泄露库中发现
   - ⚠️ 警告：该密码已泄露 N 次，建议立即更换

### 16.3 全库扫描

1. 在主界面点击「工具 → 凭据泄露检测」
2. 程序会逐一检测所有条目的密码字段
3. 检测完成后显示泄露报告
4. 可点击「跳转到条目」快速定位需要更换密码的条目

## 17. 紧急联系人恢复（v2.0 新增）

OmniKey Vault v2.0 引入了基于 Shamir 秘密共享的紧急恢复机制，允许您将主密码拆分为多份分片，分发给信任的紧急联系人。

### 17.1 工作原理

Shamir 秘密共享 (t, n) 方案：
- 将主密码拆分为 **n** 份分片
- 任意 **t** 份分片可以还原主密码
- 少于 t 份无法获得任何信息

例如：拆分为 5 份，需要 3 份才能恢复（5, 3 方案）。

### 17.2 生成分片

1. 打开「设置 → 安全 → 紧急恢复」
2. 输入主密码
3. 设置分片参数：
   - **总片数 (n)**: 2-10 份
   - **恢复阈值 (t)**: 2-n 份
4. 点击「生成分片」
5. 每份分片会显示为一串字符串，请分别保存并分发给不同联系人

> **安全建议**: 将分片分发给不同地理位置的信任联系人。不要将多份分片保存在同一位置。

### 17.3 恢复主密码

1. 收集至少 t 份分片
2. 打开 OmniKey Vault 解锁窗口
3. 点击「紧急恢复」
4. 依次输入收集到的分片
5. 当输入足够数量的分片后，主密码会自动还原
6. 使用还原的主密码解锁金库

## 24. 安全设计

### 24.1 加密架构

#### v1.6 金库（Header v2 — 双重 Argon2id 密钥拉伸）

```
主密码 (不存储)
    │
    ▼
Argon2id Round 1 (t=3, m=256MiB, p=4)  ← 第一轮：全量内存成本
    │
    ▼
MK1 (中间主密钥)
    │
    ▼
Argon2id Round 2 (t=3, m=64MiB, p=1)   ← 第二轮：二次拉伸
    │
    ▼
MK2 (最终主密钥)
    │
    ▼
HKDF-SHA256(MK2, "okv-kek-v2", salt1)   ← 域分隔符升级
    │
    ▼
KEK (密钥加密密钥)
    │
    ▼
解包 DEK (数据加密密钥)
    │
    ▼
XChaCha20-Poly1305 AEAD 加密
    │
    ▼
加密后的 Profile 负载
    │
    ▼
Ed25519 签名 → 写入 .okv 文件
```

#### v1 金库（Header v1 — 单轮 Argon2id，向后兼容）

```
主密码 (不存储)
    │
    ▼
Argon2id KDF (t=3, m=256MiB, p=4)
    │
    ▼
HKDF-SHA256(MK, "okv-kek-v1", salt)
    │
    ▼
KEK → DEK → XChaCha20-Poly1305 → Ed25519 签名
```

> **v1.6 安全升级**: 新创建的金库默认使用 Header v2（双重 Argon2id）。已有的 v1 金库仍可正常解锁和使用，系统会自动识别版本并选择对应的 KDF 路径。攻击者每次密码猜测需运行两轮 Argon2id（共 320 MiB 内存），爆破成本相比 v1 翻倍。

### 24.2 .okv 文件安全性

- **加密**: XChaCha20-Poly1305 AEAD 认证加密
- **KDF**: 双重 Argon2id（v1.6 Header v2: Round 1 256 MiB + Round 2 64 MiB = 总计 320 MiB；v1 Header v1: 单轮 256 MiB）
- **签名**: Ed25519 容器签名，防篡改
- **格式**: 自定义二进制格式，不被 hashcat / KeePass / John 等工具识别
- **域分隔**: KEK 派生标识从 `okv-kek-v1` 升级为 `okv-kek-v2`，确保 v1/v2 密钥不可互换

### 24.3 威胁防御

| 威胁 | 防御措施 |
|------|----------|
| 云盘被入侵 | 零知识加密，服务器只见密文 |
| 离线爆破 | 双重 Argon2id（320 MiB 内存成本）+ 自定义格式 |
| 源码+金库同时泄露 | v1.6 双轮 KDF 使爆破成本翻倍，域分隔符确保密钥不可互换 |
| 内存 dump | 敏感数据用完即清零 |
| 篡改文件 | AEAD 认证 + Ed25519 签名 |
| 重放攻击 | 向量时钟比较 |
| 未知设备 | 设备信任机制 + 公钥验证 |

### 24.4 内存安全

- 所有敏感数据（主密码、KEK、DEK）使用 `ReadOnlySpan<byte>` / `Memory<byte>`
- 锁定时调用 `CryptographicOperations.ZeroMemory` 清零
- 禁止业务层直接调用底层加密库

### 24.5 日志脱敏

所有日志输出经过 `LogRedactor` 过滤，自动脱敏：
- API Key 和密钥
- 密码和令牌
- 邮箱地址
- IP 地址

## 25. 常见问题

### Q: 忘记主密码怎么办？

A: 主密码无法恢复。如果您保存了恢复密钥，可以在解锁窗口尝试使用恢复密钥验证（仍需知道主密码）。如果完全没有备份，金库中的数据将无法恢复。

### Q: WebDAV 同步失败怎么办？

A: 请按以下步骤排查：
1. 检查服务器地址是否正确（以 `https://` 开头）
2. 确认使用的是**应用密码**而非登录密码
3. 点击「测试连接」验证连通性
4. 检查远端文件路径是否正确
5. 确认网络连接正常

### Q: 两台设备使用不同主密码能同步吗？

A: 不能。两台设备必须使用**相同的主密码**创建金库，因为 KEK 是从主密码派生的。如果主密码不同，远端文件的加密密钥不同，无法解密。

### Q: 同步时出现冲突怎么办？

A: 冲突会自动按「本地优先」原则解决。如需采用远端版本：
1. 使用本地文件夹同步
2. 在冲突解决向导中选择「采用远端版本」
3. 本地文件会被备份为 `.pre-sync-*` 文件

### Q: .okv 文件可以直接放到云盘吗？

A: 可以。`.okv` 文件是端到端加密的，即使云盘服务商也无法解密。您可以使用 WebDAV 同步或直接将文件放在云盘同步目录中。

### Q: 如何在多台电脑使用同一个金库？

A: 推荐使用 WebDAV 同步：
1. 在第一台电脑创建金库并配置 WebDAV
2. 在其他电脑用相同主密码创建金库
3. 配置相同的 WebDAV 信息
4. 点击同步即可拉取数据

### Q: 卸载后金库数据会丢失吗？

A: 不会。卸载只删除程序文件。金库文件 (`vault.okv`) 保存在 `%USERPROFILE%\OmniKeyVault\`，配置文件保存在 `%APPDATA%\OmniKeyVault\`。

### Q: 支持哪些平台模板？

A: 内置 66 个平台模板，覆盖国内外主流平台：

**国际平台（11 个）**：GitHub、OpenAI、AWS IAM、AWS STS、Azure、GCP、Stripe、Supabase、Anthropic、Slack、阿里云 RAM

## 18. 浏览器扩展（v1.9 新增）

> 浏览器扩展是一个**只读桥接**工具，让你在 Chrome / Edge / Firefox 中直接搜索 OmniKey Vault 保险箱里的条目，并一键复制字段值到剪贴板。

### 18.1 工作原理

```
浏览器扩展 ←HTTP→ 本地 API (127.0.0.1:14725) ←→ OmniKey Vault 桌面应用
```

- 浏览器扩展通过本地 HTTP API 与桌面应用通信
- API 仅监听 `127.0.0.1`（本地回环），**不可从网络访问**
- 所有操作均为**只读**（搜索 + 复制），不能修改或删除条目
- **敏感值永不通过 HTTP 传输**，扩展请求"复制到剪贴板"操作，由桌面应用执行

### 18.2 启用浏览器扩展 API

1. 打开 OmniKey Vault 桌面应用
2. 进入 **设置** 页面
3. 找到「**浏览器扩展**」区域
4. 勾选「**启用浏览器扩展 API**」
5. API 启动后，会显示**配对令牌**（一串十六进制字符串）

> 每次应用启动后，如果之前已启用浏览器扩展 API，会自动启动服务。

### 18.3 安装浏览器扩展

#### Chrome / Edge

1. 打开 `chrome://extensions/`（Edge 为 `edge://extensions/`）
2. 开启右上角的「**开发者模式**」
3. 点击「**加载已解压的扩展程序**」
4. 选择 OmniKey Vault 项目下的 `browser-extension/` 目录
5. 扩展图标会出现在浏览器工具栏

#### Firefox

1. 打开 `about:debugging#/runtime/this-firefox`
2. 点击「**临时载入附加组件**」
3. 选择 `browser-extension/manifest.json` 文件

### 18.4 配对浏览器扩展

1. 在桌面应用 **设置 → 浏览器扩展** 中，点击「**📋 复制**」复制配对令牌
2. 点击浏览器工具栏中的 OmniKey Vault 扩展图标
3. 点击弹窗底部的「**⚙ 配对设置**」
4. 粘贴令牌并确认

配对成功后，扩展会显示「✅ 已连接」状态。

### 18.5 使用浏览器扩展

1. 点击浏览器工具栏中的 🔐 OmniKey Vault 图标
2. 在搜索框输入关键词（条目名称、平台名等）
3. 搜索结果会显示匹配的条目及其字段（敏感字段仅显示掩码）
4. 点击字段右侧的「**复制**」按钮，字段值会被复制到剪贴板
5. 在需要填入的网页输入框中 `Ctrl+V` 粘贴

> 剪贴板会在设定的时间后自动清空（默认 8 秒）。

### 18.6 重新生成令牌

如果令牌泄露或需要重新配对：

1. 在 **设置 → 浏览器扩展** 中点击「**🔄 重新生成令牌**」
2. 复制新令牌
3. 在浏览器扩展中重新配对

### 18.7 安全注意事项

| 安全措施 | 说明 |
|---|---|
| 本地回环 | API 仅监听 127.0.0.1，外部网络不可访问 |
| Bearer 令牌 | 每个会话生成 32 字节随机令牌，扩展必须配对才能访问 |
| CORS 限制 | 仅允许 `chrome-extension://` 和 `moz-extension://` 来源 |
| 只读操作 | API 不支持任何修改、删除、创建操作 |
| 敏感值不传输 | 字段值通过桌面应用复制到剪贴板，不经过 HTTP |
| 保险箱锁定 | 保险箱锁定后，API 返回 403，扩展无法搜索或复制 |
